# Slidev Markdown Syntax Reference

Complete syntax guide for writing Slidev presentations.

## File Structure

A Slidev project needs:
- `slides.md` — your presentation content (entry point)
- `package.json` — project config (auto-generated by `npm init slidev@latest`)

Optionally:
- `components/` — Vue components (auto-registered)
- `styles/` — Global CSS
- `public/` — Static assets
- `setup/` — Vue setup scripts

## Global Frontmatter (First Block)

The first `---` block in `slides.md` is YAML frontmatter for global config:

```yaml
---
theme: default          # Theme name
title: My Presentation  # Page title
info: Description       # SEO description
author: Your Name       # Author
keywords: tag1,tag2     # SEO keywords

# Presentation config
drawings:               # Persistent drawings
  persist: false
transition: slide-up    # Default transition
controls: true          # Show navigation controls

# Style overrides
fonts:                  # Custom fonts
  sans: 'Inter'
  serif: 'Playfair Display'
  mono: 'Fira Code'
css: unocss             # CSS framework (unocss/windicss/tailwind)
---
```

## Slide Separator

Use `---` to separate slides:

```markdown
# Slide 1
Content

---

# Slide 2
Content
```

**Note:** `---` inside code blocks (```) won't create new slides.

## Per-Slide Frontmatter

Each slide can have its own YAML frontmatter:

```yaml
---
layout: cover
background: /path/to/bg.jpg
class: text-center
transition: fade
level: 1
---
```

### Available Layouts

| Layout | Description |
|--------|-------------|
| `cover` | Full-screen cover slide |
| `intro` | Introduction page |
| `default` | Standard content slide |
| `center` | Centered content |
| `fact` | Big fact/quote with description |
| `statement` | Bold centered statement |
| `two-cols` | Two equal columns |
| `two-cols-header` | Two columns with shared header |
| `image` | Full image background |
| `image-right` | Content left, image right |
| `image-left` | Image left, content right |
| `section` | Section divider |
| `quote` | Block quote |
| `none` | No default styling (full control) |

### Two-Column Layout

```markdown
---
layout: two-cols
---

# Left Column

Content on the left

::right::

# Right Column

Content on the right
```

### Image Layouts

```markdown
---
layout: image-right
image: /path/to/image.png
---

# Content on the left

The image appears on the right side.
```

## Text Formatting

```markdown
# Heading 1 (slide title)
## Heading 2
### Heading 3

**Bold text**
*Italic text*
~~Strikethrough~~
`Inline code`

[Link text](https://example.com)
![Image alt](/path/to/image.png)

> Blockquote

- Bullet item 1
- Bullet item 2

1. Numbered item 1
2. Numbered item 2
```

## Code Blocks

````markdown
```python
def hello():
    print("Hello, World!")
```
````

### With Features

````markdown
```python {1,3-4} {lines:true}
def hello():           # highlighted
    name = "World"
    print(f"Hello!")   # highlighted
    return True        # highlighted
```
````

- `{1,3-4}` — highlight lines 1, 3, 4
- `{lines:true}` — show line numbers
- `{all|2-3|1-4}` — step-through highlighting (click to reveal)

### Monaco Editor (Live Editing)

````markdown
```json {monaco}
{
  "name": "Slidev",
  "version": "1.0.0"
}
```
````

## Speaker Notes

```markdown
# Slide Content

This is visible to the audience.

<!-- This is a speaker note - only visible in presenter mode -->
```

Multi-line notes:

```markdown
# Slide Content

<!--
Speaker note line 1
Speaker note line 2
-->
```

## Click Animations (v-click)

```html
<v-click>

This appears on click 1

</v-click>

<v-click>

This appears on click 2

</v-click>
```

### Click Markers

```markdown
<v-clicks>

- Item 1 appears first
- Item 2 appears second
- Item 3 appears third

</v-clicks>
```

### Conditional Display

```html
<div v-click>Shows after first click</div>
<div v-click>Shows after second click</div>
```

## Transitions

Set per-slide or globally:

```yaml
---
transition: slide-up    # Options: slide-up, slide-down, slide-left, slide-right, fade, fade-out, zoom
---
```

## Vue Components

```html
<!-- Built-in components -->
<Tweet id="1234567890" />
<Youtube id="video_id" />
<!-- Slot annotations -->
::left::
Left content
::right::
Right content
::
```

## Global Styles

Add `<style>` blocks in `slides.md`:

```html
<style>
h1 {
  color: #3cffd0;
  font-size: 3rem;
}
.custom-class {
  background: rgba(255,255,255,0.1);
  backdrop-filter: blur(10px);
}
</style>
```

## Static Assets

- Place images in `public/` folder
- Reference as `/image.png` in Markdown
- Remote URLs work: `![alt](https://example.com/image.png)`

## Global Config File

Create `slidev/config.json` or add to frontmatter:

```json
{
  "transition": "slide-up",
  "fonts": {
    "sans": "Inter",
    "mono": "Fira Code"
  },
  "css": "unocss"
}
```

## Export Commands

```bash
# Dev server
npm run dev

# PDF export
npm run export

# PDF with notes
npm run export -- --with-notes

# PNG export
npm run export -- --format png

# Build SPA
npm run build

# Build portable (single HTML)
npm run build -- --base ./
```
